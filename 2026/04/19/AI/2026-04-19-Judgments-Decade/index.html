长琴对账·首篇：十年博客里 36 条没拖泥带水的话

为什么突然想到写这个东西呢？

有一阵子之前，和别人交流，发现自己好像对很多事情的判断还是比较准的，不过大部分都零散放在不同的博客里。于是就想，要不干脆整理出来。于是就有了本文，以及，我在博客加了一个栏目，没有公开入口，结尾有跳转。

其实我本人不喜欢预测，一定程度上比较享受不确定性，但我觉得有些趋势在当时的背景和认知下看还是比较明朗的，既然如此，何不写出来，感觉还蛮有意思的，说不定能帮到需要的人。

说起来，也是想到之前看魔兽争霸的比赛解说，尤其是看 Ted 时，他老喜欢根据战况判断结果，很多时候开场就能判断出来了。其实就是因为他非常熟悉这个游戏，能够根据初期表现大致推测未来结果。当然，也确实有说错的时候，但错也有错的道理。

这给我很大的启发，“预测”并不是重点，“分析”和“推理”才是。

一、职业的命运

1. NLP 工程师这个职业会消亡 ✅

2023-02-21 《ChatGPT 影响冲击：职业、行业与产业》

原话：“NLP 工程师不容乐观……可能并不需要那么多 NLP 工程师了。也就是说，这个职业本身正在受到严重冲击，甚至可能彻底消亡。”

写这话的时候 ChatGPT 出来才没几个月，但是当时非常有压迫感，感觉很强烈。3 年后回看，传统 NLP 任务（分类 / 实体 / 相似度）90% 被一行 LLM API 替代，“NLP 算法岗” 基本从招聘市场消失，剩下的合并成 “LLM 算法岗” 或 “AI 应用工程师”。好在，只要大家及时跟住，基本都进入更好的职业。

2. Prompt 工程师注定昙花一现 ✅

2023-02-21 同上 / 2023-10-15 《关于大语言模型的思考》

原话：“Prompt 工程以后都可能不需要了……Prompt 工程师注定会是个昙花一现的职位。”

2023 年中 “Prompt 工程师 50 万年薪” 的新闻满天飞，我说这个职位活不过 2 年。2024 起这个 title 已经从主流 JD 里消失，留下的全部叫 “应用工程师” 或 “Agent 工程师”。这个判断当时是非常肯定的，我举过例子，比如之前也没有 word 工程师、excel 工程师，这是一个道理。

3. 未来只剩两种算法工程师：AI 算法 + 传统 ML ✅

2023-02-21 同上

原话：“未来，可能只有 AI 算法工程师和传统的机器学习算法工程师这两种算法工程师。”

NLP / CV / 语音三大算法岗在 2024-2026 几乎完全合并为"多模态 LLM 算法岗"。专精某一模态的简历在大厂面试里已经不占优势，反而可能被认为"路径太窄"。

4. 算法工程师未来主要做"LLM 之外的剩余 10%" ⏳

2024-12-20 《关于 AI 前沿的思考》

原话：“以现有 LLM 的实力，基本能解决 80% 的问题，有时候甚至是 90%，但后面这 10% 恐怕只靠 LLM 就不够了。”

2025-2026 年正在出现的"业务工程 / 多 Agent 编排 / RL 后训练 / 推理加速"四条新岗，本质上就是 LLM 之外的剩余 10%。这 10% 反而是算法工程师真正的护城河——前 90% 已经被 API 平民化了。

5. 算法工程师必须先是个工程师 ✅

2025-02-17 《DeepSeek R1 深度技术解析》

原话：“对算法工程师，尤其是 LLM 相关的算法工程师来说，只懂算法怕是难以应付以后的局面；退一步说，算法工程师你不也得先是个工程师么。”

2025 起所有头部公司的算法岗 JD 都加了一句"工程能力优先"。能跑通 vLLM / SGLang、能改 verl 一行代码、能 debug 一段 CUDA Kernel，比能背 attention 推导更值钱。

二、技术的下一招

6. AI = 先验 + 演绎 + 归纳 三件套（九宫格框架）✅

2018-07-22 《NLP 与 AI》

原话（提炼）：知识图谱（先验）× 演绎规则 × 神经网络（归纳）三层 × 通用 / 类别 / 具体 三层，构成 AI 的"九宫格"。

写这话时 BERT 都还没出。8 年后，LLM（归纳）+ RL（演绎）+ RAG / 知识图谱（先验） 正是当前主流大模型架构的三件套。这是我自己最得意的早期判断——不是因为多准，而是因为框架性的判断比单点预言更稀缺。

7. 多模态是 NLP 必然方向（伏笔于 2018，明确于 2021）✅

2018-09-19 伏笔 《NLPFA Ch01 NLP 的方向》 已把"多模态"列为 NLP 的子方向之一（大纲位置）。

2021-10-27 明确判断 《Senta：百度情感分析模型》：

“未来一定是围绕着更深刻的语义理解发展的……多模态：多种不同形态的输入结合……换成机器，自然也可以做类似的事情，预期来看，效果必然是有提升的。”

2018 写 NLP 大纲的时候就已经预留了"多模态"位置；2021 年圈内主流还在拼 BERT 微调和 Prompt 的时候，把方向落成判断。两年后 GPT-4V 出来，又两年 Gemini 1.5 / Sora / Veo / 端到端语音。多模态成了所有 LLM 进步的内核——这条不是"硬预言"，是把方向押得早。

8. 句子表征下一次创新会从"段到句"🔄

2022-03-27 《句子表征综述》

原话：“也许，下一个创新会从段到句？毕竟，当我们看一段话和一句话时模式看起来好像并不相同。让我们拭目以待。”

2022 年大家还在卷 SimCSE。我说下一招得跨"段"和"句"的尺度差。3 年后 long-context（128K → 1M → 10M）、document-level reasoning、RAG-over-document 全面爆发。部分中——尺度跃迁这件事我押对了，具体形态不是我想的那种"段→句"对称建模，而是被 long-context Transformer 直接吃掉。

9. 强化学习是新方向，下一招"先思考再生成"✅✅

2023-02-21（同日草稿，未发布）《ChatGPT 实现流程》

原话：“强化学习是新方向。Prompt tuning 那一套太不优雅了，优雅是程序员毕生的追求。下一个方向是什么？……先思考再生成？”

写在草稿里的 25 个字。18 个月后 OpenAI o1 发布——其核心范式正是 thinking-then-answering；23 个月后 R1 / R1-Zero 把它拍成行业标配。这是目前发现的最早一条对 reasoning paradigm 的明确预言，比下面 #11 那条"o1+RL"还早 22 个月。草稿里的话也是话，时间戳不会骗人。

10. DPO 和 PPO 不在一个维度上 ✅

2023-10-15 《关于大语言模型的思考》

原话：“我个人一直没有关注过 DPO，就是觉得它和 PPO 不在一个维度上，没有可比性。当然，我更加认同 PPO 的设计思想。”

写这条的时候 DPO 是全网最香的 RLHF 替代方案，paper 引用一个月 + 1000。我说"它和 PPO 不在一个维度"。R1 / Kimi-K1.5 / DAPO / GRPO / GSPO / CISPO ……过去 18 个月所有真正改变 SOTA 的算法，全部走 PPO 系，没有一个走 DPO。DPO 至今主要做对齐辅助，不做核心 RL。

11. o1 + RL 是不被很多人重视、但很可能带来下一次革命的组合 ✅

2024-12-20 《关于 AI 前沿的思考》

原话：“它（o1）和强化学习的结合应该会是一个不被很多人重视（或看到），但很有可能带来下一次革命的组合。”

写完这话 60 天后 R1 发布，全行业转向 RL。这是我过去 5 年里最准的硬预言之一——不是因为我聪明，是因为我从 2018 年就一直押 RL，看 o1 时第一眼就看到 RL 的影子。坐了 7 年的冷板凳，R1 出来时可能没有任何一个写综述的人比我心情更复杂。

早期方向预判（更早伏笔）：2023-01-25 《ChatGPT Prompt 工程》 已经写过：

“关于推理，也许 Model Cascades、CoT（Chain-of-Thought）等会发力，但我觉得也不能排除符号 AI 的再度兴起。”

——2023-01 就把"推理"作为 ChatGPT 之后的下一突破点。比 o1 发布早 20 个月。

12. R1 之后所有 LLM 都会"R1"一下 ✅

2025-02-17 《DeepSeek R1 深度技术解析》

原话：“我想，后面所有的 LLM 可能都会’R1’一下的。”

写完 6 个月后：Qwen3 R1 一下，Kimi-K2 R1 一下，GPT-5 R1 一下，Gemini-2.5 R1 一下，Claude 4.5 R1 一下，豆包 1.5 R1 一下。一句"R1 一下"成了 2025 年的行业 verb。

13. DLM 的关键根本不是并行，而是模式 ⏳

2026-03-01 《DLM 漫谈》

原话：“DLM 的关键根本不是性能（并行），而是模式！我个人总是感觉 MDLM 看起来有一点点怪怪的，感觉硬套 AR 模式……所有强行将 DLM AR 化的做法都感觉怪怪的。”

DLM 这一波的主流叙事是"我们把 AR 加速了 N 倍"。我的判断反过来：把 DLM 当"更快的 AR"才是浪费。DLM 真正的价值在于"非线性 / 可中断 / 全局感知"的思考模式。1 个月后我写的 《Think Strategy》里 latent thinking 论文密度急剧上升，trend 在我这边——但还没到能立 ✅ 的时候。

14. OpenClaw 之上必然出现"Agent of Agents"编排管理层（首发于 2018）⏳

2018-07-12 愿景种子 《NLP 业务与系统》：

“希望未来每个人有自己的机器人……每个企业都应该有点餐机器人……我跟手环说「帮我点个巨无霸吧」，它知道这是麦当劳的产品，会找到麦当劳的机器人，两个机器人对话，帮我搞定。”

2024-02-03 升级为明确判断 《LLM Tiny Pretrain》：

“多 Agent 智能，多个能完成简单功能的小模型组合成复杂的系统……笔者觉得这个方向会更加关注’组合’，而不是’小模型’，'小模型’只是方法，有机’组合’才是目的。”

2026-03-13 OpenClaw 视角具象化 《从 OpenClaw 到 AI Coding》：

“未来一定会出现多个 OpenClaw 之上的一个管理界面，让用户更加方便的去很容易的创建多个不同风格的 OpenClaw……剩下的底层的都是通用的。”

8 年前就在 NLP-Business-and-System 里画过"两个机器人对话"的未来——当时还是 BERT 都没出的时代，纯属愿景。2 年后（2024-02）小模型语境下把它升级成"组合是目的"的明确判断。4 年后（2026-03）OpenClaw 出现后给出具体落地形态。不是 Coze / Dify 那种单实例 builder——而是真正的"Agent of Agents"统一管理面板。可证伪条件：到 2027-12-31，仍未出现一个被广泛使用（GitHub star ≥ 10K 或商业 DAU ≥ 100 万）的多 Agent / 多 OpenClaw 编排管理产品，本判断需要修正。

三、行业的形状

15. 当时火热的"AI 客服"基本是模板分类器（写于 2019）✅✅

2019-07-20 《ChatBot 设计》

原话：“去看了一下发现虽然没有什么新东西（意图理解是一个分类器，多轮对话是设计好的模板，甚至对话的生成也是基于模板）……感觉还不如三年前 memect 的知识图谱小组。冷静点看，现在的人工智能其实过于依赖数据……人一旦对某一事物开始投入就会逐渐变得盲目。”

2019 年"AI 客服 / 智能机器人"是大众媒体宠儿，估值动辄独角兽级。2023 后 LLM 普及，那一代"伪 AI 客服"几乎全部被淘汰或重做。这条不是预言，是反共识吐槽——但它让我从 2019 起就避开了那个赛道泡沫。敢于说"皇帝没穿衣服"也是判断。

16. 个人或几个人能做以前成百上千人的事 ✅

2023-02-21 《ChatGPT 影响冲击》

原话：“个人或几个人能够做的事情甚至可比此前成百上千人的团队。”

写这条时 GPT-4 还没发。3 年后，Cursor / Bolt / Devin / V0 / Trae / 各种"超级个体公司"已经把这条直接验证到刺眼——10 万 ARR 的 SaaS 后面常常只站一个人。3 年后亲自印证：2026-01-01 《以 AI Coding 之管窥探世界之变》 写道：“AI 超级个体时代即将来临！……我也已经看到很多个人已经完成了数十万行代码的复杂项目并且上线。”

17. AI 研究和应用会两极化、中间层算法工程师在夹缝中求生 ✅

2023-02-21 同上

原话：“AI 研究和应用会越来越两极化……能干这事儿的没几家，能干出来的可能更少……应用侧却由于 LMAS 而更加平民化，这就导致了中间的算法工程师有点尴尬。”

国内能做基模的公司收敛到 5-7 家（DS / Qwen / Kimi / 智谱 / 字节豆包 / Minimax / 阶跃），夹在中间"调 BERT、做特征工程"那批人在经历转岗潮。这条判断的残忍程度在 2023 年初被严重低估——包括我自己。8 个月后再深化：2023-11-04 《OpenAIGC》 把这条心理化了——“算法研究的聚集化和算法应用的普及化……两头都卷，我们却不得不选择走向一头”，这是同一个判断的"亲身处境版"。

18. LLM 作为外部"最强大脑"会被每个人轻易获取 ✅

2023-04-15 《ChatGPT 介绍》

原话：“LLM 作为一个外部「最强大脑」未来一定会非常容易被每个人获取，至于用来做什么，Depends Your Imagination。”

2024-2025 ChatGPT app / 豆包 / Kimi / DeepSeek / Claude / Gemini 全部移动端化、免费化。2026 国内有上亿 LLM 月活用户。"非常容易"已是事实——而 2023 年 4 月写这话的时候，国内绝大多数人还没用过 ChatGPT。

19. LLM 会成为像电、像 K8S 一样的 Infra（首发于 2023-02）✅

2023-02-21 首发（"新电力"类比） 《ChatGPT 影响冲击》：

“AI 一定会深入到全社会每一个领域，就好像现在无处不在的电一样……（吴恩达说 AI 是新的电力）但恐怕连他都没想到这么快就能看到了。”

2023-10-15 具象化（"K8S 容器"类比） 《关于大语言模型的思考》：

“大模型正在成为 Infra，而且这个过程非常迅速。可以预计，用不了几年，大模型就和 K8S、容器一样普遍了。”

2023-02 写"新电力"的时候，国内大厂的大模型 API 还没几家上线——8 个月后再用"K8S 容器"做更具体的类比，这中间是越来越笃定的过程。2024-2026 LLM API 已是所有应用的默认 dependency。pip install openai、pip install anthropic 已经像 pip install requests 一样常见。Infra 化的速度比当年 K8S 还快。

20. 不会有某个突然的 AGI 时刻、AGI 是分形过程 ⏳

2024-12-20 《关于 AI 前沿的思考》

原话（引 Bob 并完全认同）：“不会有某个突然的 AGI 时刻，这些问题是分形的，我们会看到越来越多的自动化，但未来可能会很平凡。”

写这话之后 16 个月——确实没有"GPT-5 引爆 AGI"那种戏剧时刻。反而是 R1 / Sonnet 3.5 / Cursor / Veo 3 / o3 / Sora 2 这种分形进步。每一个都是大新闻，但没有一个让你感觉"明天起世界变了"。

21. 主动性将成为稀缺生产要素 ⏳

2024-12-20 同上

原话：“从一个智力是稀缺资源的世界转向一个智力将变得普遍且免费的世界，主动性将成为稀缺的生产要素。”

写完 16 个月，已经能在身边看到信号：人和人的差距不再是"会不会用 AI"，而是"想不想用、敢不敢决定要做什么"。"对自己有要求"这件曾经被当作软技能的事，正变成最硬的硬技能。

22. MCP 从技术层面没什么，热闹一阵会冷 ✅

2025-05-01 《Seed Thinking & Qwen3》

原话：“对于 MCP 我应该算是国内最早知道的一批人……看到 MCP 后，说实话，从技术层面并没有觉得它有什么……那次沟通完一个月后，MCP 火了。不过个人依然无感。”

写这话的时候 MCP 正爆火，所有 LLM / Agent 平台抢着接入。1 年后回看（2026），MCP 已经从顶峰回落——“MCP everything"的 hype 退潮，主流回归 Function Calling / 自家 Agent 协议。这是一条反 hype + 短窗口的判断，比"押对方向"难——因为得在所有人都在追的时候说"没啥”。

四、科学的方向

23. AI 与 IA 双线并进，奇点不可知（写于 2016）✅

2016-03-31 《与机器人共舞》读后感兼谈 AI 与 IA

原话：“我们大胆预测，AI 和 IA 依然将在各自领域不断向前，我们不知道奇点几时来临，唯一能做的只有在它到来之前做好充分的准备。”

这是我在博客上最早的、可以挂年限的预言。十年回看：AI（自主智能 LLM、Agent）和 IA（增强智能 Cursor、Copilot、各种插件）的确在并行发展且彼此渗透。"奇点不可知"也对——和 #20 那条"AGI 是分形过程"完全一脉相承，只不过提前 8 年。

24. 我们关注的从来不是信息，而是共鸣 ⏳

2017-09-07 《语言、AI、情感》

原话：“我们关注的从来不是信息，而是共鸣。”

写这话的 2017，"AI 陪聊"在国内还是被嘲笑的方向。9 年后 Character.ai / Talkie / Replika / 各种 Roleplay 产品爆发，月活成亿。共鸣是底层需求，信息只是壳——这个判断需要的不是数据，是阅历。

25. 知识足够多就有可能产生"智慧"（机器同理）✅

2020-11-28 《分类与 AI》

原话：“我们应当可以大胆预测，当一个人拥有足够多的知识时，他就有可能产生’智慧’；那么同样，当一个机器拥有足够多的知识时，他也有可能产生’智慧’。”

写这话时 GPT-3 paper 出来才半年，"涌现 / emergence"这个词还没在大众语境里出现。两年后 ChatGPT 验证了"足够多的知识 → emergent capabilities"这条路径。Scaling Law / 知识压缩 / emergence——这条 2020-11-28 的"大胆预测"是工程派与哲学派两边都中。

26. 下一突破点是世界模型 + 实时学习（首发于 2023）⏳

2023-01-20 首发 《GPT-3 论文笔记》：

“未来的方向包括：从人类学习目标函数、强化学习微调或多模态（比如更好的世界模型）。”

2025-03-15 推进 《LLM 应用开发》：再次提到具身智能、世界模型方向。

2025-08-11 系统化 《群聊中的 AGI 拼图》：完整论述"世界模型 + 实时学习是下一突破基础"，包含"知道自己知道或不知道"的元认知、Sutton “去中心化神经网络”、知识编辑等。

这条最早写在 2023 年 1 月——ChatGPT 发布后 2 个月，全行业还在惊讶 GPT 能力的时候，我已经在 GPT-3 笔记里把"世界模型"作为下一方向写下来了。2025-08 系统化展开。2026 年 World Model + Online Learning 论文密度急剧上升，trend 在路上。可证伪条件：如果 2027 年底世界模型 + 实时学习两条线仍没出现工业级明显进展，承认这条判断有问题。

27. 视频 AI 两年内做出"人想看的电影"是过于乐观 ⏳

2024-12-20 《关于 AI 前沿的思考》

原话（反 Bob 的预言）：“（Bob 觉得 2 年内 AI 可以创作出人们真正想看的电影）我觉得可能会被打脸。”

2026 年已过半，Veo 3 / Sora 2 在 30 秒短片层面接近"可看"，但"人们真正想看的电影"层级仍未达到。这是我给 Bob 的一记反向预言——目前看我赢面更大。Bob 的逻辑是 Scaling，我的逻辑是"故事是判断、不是预测"。

28. 具身智能 2-3 年元年（不需 5 年）⏳

2024-12-20 同上

原话：“Bob 认为 5 年内能看到明显进展，我个人认为不用 5 年，可能也就 2-3 年。”

2025 起 Figure / 1X / 智元 / 宇树等动作密集，2026 已被业界普遍当作"具身智能元年"。两次反 Bob 的判断里，我赢的概率更大——不是因为我比他懂，是因为我离 implementation 更近。

29. 2025 是语音 Agent 爆发年 ✅

2024-12-20 同上

原话：“可以比较确定地看到，2025 年将是语音 Agent 爆发的一年。”

2025 一年内：OMNI 全面普及、ElevenLabs / SesameAI 实时语音爆发、字节豆包语音、阿里 Cosyvoice、Kimi 语音助理。一年时间窗的具体预言能命中，靠的不是聪明，是已经在那条线上做了一年。

五、个人的方法论

30. 真正的稳定 = 任何时候都能找到工作 ✅

2021-02-19 《AI 工程师成长 I》（5 年后在 2026-01 《ListenGlimmer 000》 又重述一次）

原话：“真正的稳定应该是任何时候都能找到工作，而不是在一个固定的工作岗位上待着不动。在某个稳定的环境下呆久了，我觉得反而是一种巨大的不稳定。”

2024-2026 互联网寒冬 + LLM 冲击叠加，"待在大厂 = 稳定"叙事彻底崩塌。这条 5 年前写下的话让我在 2025 年敢做超级个体、敢把"个人能力"作为最大资产。穿越 5 年仍稳定持有的判断，比当下灵光更稀缺。

31. 任务 > 工具

2023-04-22 《ChatGPT 开发指南》

原话：“我们始终强调’任务’多于’工具’，ChatGPT 是目前最好的工具，但未来一定会有其他 LLM 出现。但只要我们理解了要做的事情，理解了系统设计，工具就能为我所用。”

3 年里 LLM 换了 10 茬，但任务的形状基本没变。这不是判断，是一种习惯——但它是其他所有判断之所以站得住的底层操作系统。

32. 复现是唯一的认知验证手段

2025-02-17 《DeepSeek R1 深度技术解析》

原话隐含：“看到 R1 论文的第一反应是不可能吧？自己跟着复现后真的是震惊到了，太漂亮了。”

R1-Zero 自己跑过、做了三组消融、看到 reward 真的爬起来——之后写的判断和没复现过的人写的判断，分量不一样。这条不是预言，是方法论的元判断：别人的总结不能替你的复现，二手判断没有重量。

33. 网上已经有了的，就索然无趣，基本不会再碰

2025-12-22 Diary 等多处

原话：“如果网上已经有了，就索然无趣，基本不会再碰。”

这条原本是写感情、被我复用为选题原则。它解释了为什么我不写"GPT-5 全网最快解读"、不写"Cursor 1.0 三十个用法"——已经有人写了的事，挪去做这件事的边际效用基本为零。

34. 大势面前个人努力微不足道，但仍要踩着浪花前进

2026-01-01 《以 AI Coding 之管窥探世界之变》

原话：“大势面前个人的努力微不足道。那怎么办，只能踩着浪花前进了啊。”

这是我作为一个 INTJ 在 2026 找到的妥协解。承认渺小、不放弃前进。古典派写这种话不带矫情。

35. 使命与生命同样重要

2026-02 个人草稿（未发布）

原话：“我相信一个人的使命和生命同样重要。”

这条不在博客里，写在我自己的"练级"草稿里——比任何对外文案都更接近本心。它解释了我 2024-2026 一系列"反主流"决定（拒读博、不去大厂躺平、坚持开源教程）的根因。草稿里写下来的句子，往往比公开发表的更值得被记住。

36. 不读博，但要把博士那部分获益拿到手

2026-04-18 私密草稿

原话：“不要脱产读博。损失 3-5 年时间换一张证书，对我来说收益太低……争取一个’非全博士 / 联合培养博士’，或拒绝学位但建立’独立研究员’身份。”

这条仍是当下判断，5 年后回看才能验证。但它是我目前为止做得最反主流的职业选择——周围 30+ 的算法工程师里，"想读博/正在读博"的比例不低。我的逻辑很简单：你已经在过博士生活，你只是没有学位。

接下来 3 年我打算继续验证的 5 条新预言

给自己立旗，给读者打靶。如果到 2029 年这 5 条都还没成真，欢迎回来打脸。

预言 A（技术）：2027 年底前，会出现一个 latent thinking 路线的明确 SOTA

具体可证伪条件：到 2027-12-31，仍没有一个 reasoning 模型在 AIME / Math-500 / GPQA-Diamond 三个 benchmark 上同时超过同尺寸的 token-CoT 模型 ≥ 5 个百分点的，本预言落空。

预言 B（技术）：到 2028 年，会出现一个工业级的实时学习 / 持续学习方案

具体可证伪条件：到 2028-12-31，仍没有任何主流 base model（参数 ≥ 7B）支持"在线学习单个用户偏好且不灾难性遗忘"作为开箱即用功能，本预言落空。

预言 C（行业）：2027 年开始，"Agent 数据 / RL 范式工程师"会成为 LLM 公司的硬岗位

具体可证伪条件：到 2027-12-31，国内主流 LLM 公司（DS / Qwen / Kimi / 字节 / 智谱）的招聘 JD 中，仍未出现独立标题的"Agent Data Engineer"或"RL Pipeline Engineer"岗位，本预言落空。

预言 D（行业）：到 2028 年，"个人超级助理"产品形态会从演示级走到日活级

具体可证伪条件：到 2028-12-31，仍没有任何"个人超级助理"产品（非工具栏目，需含主动 / 长期记忆 / 跨设备）单 App DAU 超过 5000 万，本预言落空。

预言 E（自我）：到 2029 年，我会写出一本叫《Agentic RL 时代的训练范式与数据工程》的书

具体可证伪条件：到 2029-12-31，仍未出版（任意形式，含电子书 / 付费专栏）此书，本预言落空。

收尾

我并不是想说这些判断有多准——里面有运气、有偏差、有事后追认的味道。我想说的是另一件事：

判断的密度和细度，是判断者唯一的护城河。 一个写过 298 篇博客的人，跟一个看了 298 篇博客的人，看着同一条新闻时看到的是不一样的东西。前者从 2018 年就在押 RL 这件事本身，会让 2024 年那条 o1+RL 预言的命中率从 50% 飙到 90%。

我把判断集中拎出来，是给自己一个底气，敢继续在下一个 5 年里继续判断。也是给读到这里的你一个邀请：自己回头翻翻你 5 年前写的东西，找出那些拍着胸脯不带"也许"的话——那才是你的资产，比你 GitHub 的 star 数有意义得多。

关于"对账"这件事

这一篇是 v1。本意不止是"拎出 35 条得意话"——是想给自己立一份长期账：判断要算账。

所以从今天起，长琴博客有了一个常驻页面：长琴对账簿 ——以时间线 + 主题筛选 + 验证状态色标的方式持续维护这份名单。每年 / 每个关键节点回头补几条、验证几条；对的标 ✅，错的也老实标 ❌。

判断要算账，账要敢算。这是判断者唯一的尊严。

不追风口，只追长期。

用十年的尺度看一年的事。

大模型越来越趋向于做正确的事，有标准的事，其他方面就不太会做了，没有标准，难以评判，众人评判不一。做神，天地不仁以万物为刍狗，天若有情天亦老。

更加需要后训练和对齐，去补充大模型不足。
